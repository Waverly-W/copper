# 剪贴板管理插件 PRD

## 1. 文档信息

- 产品名称：uTools 个性化剪贴板管理插件
- 文档版本：v0.1
- 文档状态：草案，已基于当前需求访谈收敛
- 适用范围：MVP 至 v1.0 规划输入

## 2. 产品背景

官方剪贴板插件无法满足个人高频工作流，主要问题在于：

- 无法充分覆盖日常复制内容形态
- 收藏与历史记录工作流割裂
- 键盘效率不足
- 搜索和定位效率不够高
- 交互无法针对个人习惯深度定制

目标是基于 uTools 构建一款偏个人使用、完全本地优先、键盘优先的剪贴板管理插件，替代官方剪贴板插件，覆盖“收集历史记录 + 快速检索 + 收藏沉淀 + 一键回填”四类核心场景。

## 3. 产品目标

### 3.1 核心目标

- 尽可能完整记录用户日常复制内容
- 在单页中同时承载历史区与收藏区，不切页
- 让高频操作可完全通过键盘完成
- 在 5000 到 10000 条记录规模下保持极快搜索体验
- 所有数据仅保存在本地，不由插件主动通过网络传输

### 3.2 成功标准

- 用户可以通过插件找到最近至少 30 天内的大多数复制内容
- 用户可以在不使用鼠标的情况下完成搜索、定位、复制、粘贴、收藏、删除
- 用户能把历史记录沉淀为可长期维护的高频片段库
- 插件在文本、图片、文件三类 MVP 内容上形成稳定闭环

## 4. 产品定位

这不是一个“展示型剪贴板列表”，而是一个偏生产力、偏键盘流、偏个人资料沉淀的剪贴板工作台。

产品定位特征：

- 键盘优先
- 双栏工作流
- 本地优先
- 性能优先
- 收藏可运营
- 历史不可篡改

## 5. 目标用户

当前阶段目标用户仅为单一用户，即插件作者本人。

用户特征：

- 高频复制粘贴内容
- 需要同时处理文本、代码、图片、文件等多种内容
- 更依赖键盘而非鼠标
- 对性能、密度和操作路径敏感
- 不需要复杂的团队共享、同步和权限体系

## 6. 使用场景

### 6.1 历史检索

用户复制过一段文本、代码、图片或文件后，希望稍后通过搜索快速重新找到它。

### 6.2 快速回填

用户打开插件，定位到某条记录，直接将其粘贴回打开插件之前的输入框、编辑器或目标窗口。

### 6.3 高频片段沉淀

用户将常用回复、命令、模板、代码片段等从历史区加入收藏区，并按 Tab 管理。

### 6.4 文件与图片重用

用户复制过一张图片或某个文件后，希望再次快速复制或粘贴，无需重新定位原内容来源。

## 7. 范围定义

### 7.1 MVP 范围

MVP 必须覆盖以下能力：

- 后台持续监听系统剪贴板
- 支持 Windows 与 macOS
- 历史区与收藏区同页左右分屏
- 支持文本、图片、文件、文件夹、多文件复制的记录、检索、复制、粘贴
- 搜索框全局置顶，默认获取焦点
- 收藏区支持 Tab 管理
- 历史项支持加入收藏
- 收藏项支持编辑
- 提供基础设置页

### 7.2 v1 后扩展范围

- HTML / 富文本 / Office / WPS / Excel 内容的更高保真支持
- 拼音搜索优化
- 代码块语法高亮渲染
- 更强的数据清理、压缩与索引策略
- 更细粒度的展示配置

### 7.3 非目标

以下内容不属于当前阶段目标：

- 网络同步
- 来源应用识别与展示
- 类型筛选、时间筛选、来源筛选
- 多用户协作
- 隐私打码、黑名单、无痕模式
- 富文本视觉样式的 100% 还原

## 8. 设计原则

- 单页完成主要任务，不做页面跳转
- 优先确保高频路径最短
- 优先保证文本、图片、文件闭环可用
- 优先保证性能，再考虑信息丰富度
- 历史记录强调真实性与可追溯，收藏强调可运营与可维护

## 9. 功能需求

### 9.1 剪贴板监听

#### 目标

持续监听系统剪贴板变化，在插件未打开时也能记录新增复制内容。

#### 要求

- 必须支持后台持续监听
- Windows 与 macOS 第一版都需支持
- 监听到新复制内容后，自动完成归档
- 插件不主动将记录发送到网络

#### 支持内容形态

- 纯文本
- 代码块
- HTML / 富文本
- 图片
- 文件
- 文件夹
- 多文件复制
- 浏览器里复制的图片
- 截图工具复制的图片
- Office / WPS / Excel 单元格复制内容

说明：

- 上述形态是产品目标范围
- MVP 首先保证文本、图片、文件链路稳定
- 其余富文本类内容允许分阶段完善

### 9.2 历史记录

#### 目标

保留足够长的复制历史，并支持快速检索与再次使用。

#### 要求

- 至少覆盖最近 30 天的记录
- 同时支持 5000 到 10000 条规模的历史容量
- 文本类内容按内容去重
- 相同文本内容合并为同一条历史项，并累计：
  - `copyCount`
  - `lastCopiedAt`
- 图片与文件不参与文本去重逻辑
- 历史项不可编辑
- 历史项支持删除
- 历史项支持加入收藏

#### 文件失效策略

- 文件、文件夹、多文件复制历史仅保存路径与必要元数据
- 若源路径失效，允许记录失效并在界面标记不可用
- 不要求对文件内容做快照

### 9.3 收藏区

#### 目标

提供高频片段库能力，而非仅做历史项快捷引用。

#### 要求

- 收藏区与历史区在同一页面左右分屏展示
- 收藏区顶部使用 Tabs 管理分类
- 同一收藏项可被多个 Tab 引用
- 收藏项可独立存在，不依赖原历史项生命周期
- 收藏项允许编辑
- 收藏项支持删除

#### 删除语义

- 当收藏项仅存在于一个 Tab 引用中时，删除视为删除该收藏项本体
- 当收藏项存在于多个 Tab 中时，删除默认删除收藏项本体

备注：

- 当前删除语义来自现阶段需求确认
- 该设计后续可能需要在可用性测试中再次验证，避免误删

### 9.4 搜索

#### 目标

在大规模历史数据下，提供极快的关键字定位能力。

#### 要求

- 搜索框位于页面顶部，默认聚焦
- 搜索范围同时覆盖历史区与收藏区
- 历史区与收藏区分别异步返回结果
- 两侧结果互不影响
- 默认键盘焦点优先进入历史区
- 可通过方向键快速切换到收藏区
- 搜索不支持类型筛选、来源筛选、时间筛选
- 排序逻辑以匹配命中优先，辅以时间因素

#### 搜索能力要求

- 支持关键字模糊匹配
- 支持拼音搜索中文文本
- 支持匹配文件名
- 支持匹配图片关联名称或存储名
- 支持高亮命中片段

#### 性能目标

- 在 10000 条记录规模下，搜索结果应接近毫秒级返回
- 搜索过程不得因 hover 信息加载影响主线程输入体验

### 9.5 复制与粘贴行为

#### 单击行为

- 单击任意列表项，重新复制到系统剪贴板

#### 粘贴行为

- 粘贴目标为打开插件前的活动窗口
- 文本粘贴为文本
- 图片粘贴为原图
- 文件粘贴为文件引用
- 粘贴完成后立即隐藏插件并返回原窗口

#### 键盘定义

- `Ctrl/Cmd + F`：聚焦搜索
- `Tab`：在历史区和收藏区之间切换
- `↑ / ↓`：在当前列表内移动
- `→`：从历史区快速切换到收藏区
- `Enter`：复制当前项
- `双击 Enter`：粘贴当前项并关闭插件
- `Ctrl/Cmd + 1~9`：切换收藏 Tab
- `Ctrl/Cmd + D`：删除当前项
- `Ctrl/Cmd + S`：收藏 / 取消收藏

### 9.6 列表展示

#### 总体布局

- 上方为搜索区
- 下方为左右双栏
- 左侧历史区
- 右侧收藏区
- 布局比例固定为 5:5

#### 展示策略

- 界面风格紧凑
- 列表默认只展示复制项主体内容
- 时间、字符数等附加信息通过 hover 展示
- hover 相关信息应延迟或异步加载，不阻塞主列表渲染
- 不提供独立预览区

#### 长文本

- 默认展示最大 6 到 8 行
- 超出部分折叠
- 支持展开与再次折叠

#### 代码块

- 内容按代码块语义展示
- 语法高亮属于增强项，不作为 MVP 必达项

#### 图片

- 设定最大展示高度
- 小于最大高度时按实际大小显示
- 大于最大高度时按限制高度缩放

### 9.7 设置页

#### MVP 必须提供的设置项

- 主题模式：
  - 浅色
  - 深色
  - 跟随系统
- 历史记录保留上限
- 图片最大展示高度
- 长文本折叠行数
- 默认聚焦和键盘行为配置
- 是否后台监听

## 11. 信息架构

页面层级建议如下：

- 主页面
  - 顶部搜索区
  - 左侧历史区
  - 右侧收藏区
- 设置页

主页面是绝对主路径，设置页属于辅助路径。

## 12. 交互流程

### 12.1 搜索并复制

1. 打开插件
2. 搜索框自动聚焦
3. 输入关键字
4. 历史区和收藏区异步刷新结果
5. 默认焦点进入历史区结果
6. `Enter` 复制选中项

### 12.2 搜索并粘贴

1. 打开插件
2. 输入关键字
3. 使用方向键选中目标项
4. 双击 `Enter` 执行粘贴
5. 插件关闭
6. 内容回填到打开插件前的目标窗口

### 12.3 将历史项加入收藏

1. 在历史区定位到目标项
2. 执行收藏快捷键
3. 选择一个或多个 Tab
4. 收藏项写入收藏区

### 12.4 编辑收藏项

1. 切换到收藏区
2. 选中目标项
3. 进入编辑
4. 修改标题或内容
5. 保存后影响所有引用该收藏项的 Tab

## 13. 性能要求

- 10000 条规模下保持搜索输入流畅
- 列表滚动无明显卡顿
- hover 信息加载不得阻塞主列表交互
- 图片缩略展示不得阻塞初次列表渲染
- 搜索结果应优先由本地索引返回，而非每次全量扫描

## 14. 平台与技术约束

### 14.1 平台约束

- 需运行于 uTools 插件环境
- 需支持 Windows 与 macOS
- 粘贴能力依赖 uTools 提供的输入相关 API

### 14.2 技术风险

- 通用系统剪贴板后台监听的官方能力边界需要进一步验证
- HTML / 富文本 / Office / WPS / Excel 的高保真采集与回填风险较高
- 跨平台在图片、文件、富文本粘贴行为上可能存在系统差异
- 本地数据库与图片落地文件的协同清理需要专门设计

## 15. 实现建议

### 15.1 阶段划分

#### 阶段一：基础骨架

- 搭建主页面双栏布局
- 搭建设置页
- 搭建历史、收藏、Tab 基础数据模型
- 完成键盘导航主流程

#### 阶段二：MVP 闭环

- 打通文本、图片、文件的监听、存储、搜索、复制、粘贴闭环
- 完成图片私有目录落地
- 完成历史加入收藏能力

#### 阶段三：性能优化

- 建立本地搜索索引
- 优化列表渲染
- 优化 hover 懒加载信息
- 完成拼音搜索支持

#### 阶段四：富文本增强

- 增强 HTML / 富文本的结构化存储
- 评估 Office / WPS / Excel 内容的可保真方案

## 16. 验收标准

MVP 达标需满足以下条件：

- 可以后台记录系统剪贴板变化
- 文本、图片、文件三类内容可稳定记录
- 历史区与收藏区可在同页双栏使用
- 收藏区支持多 Tab
- 历史文本支持去重累计次数
- 单击可复制，双击 `Enter` 可粘贴并关闭插件
- 搜索框默认聚焦
- 键盘主流程可独立完成主要任务
- 设置页可配置基础展示和行为参数

## 17. 待确认与开放问题

- uTools 插件环境下后台监听剪贴板的最佳实现方式
- 富文本与 HTML 的可保真存储结构
- Office / WPS / Excel 内容在跨平台下的统一抽象方式
- 收藏删除语义是否需要“仅移除当前 Tab 引用”和“删除实体”双操作区分
- 是否需要为失效文件记录提供统一修复入口
